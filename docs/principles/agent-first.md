# Agent-First 工程原则

本项目遵循"人类掌舵、代理执行"。

## 核心原则

1. 人类时间最稀缺：优先投入在环境、约束、反馈回路，而不是手工重复编码。
2. 约束优先于提示：把规则写进 lint、测试、hook、CI，减少靠记忆执行。
3. 渐进披露：入口文档保持简短，细节拆分到专题文档，避免上下文过载。
4. 可读性即能力：代理可直接读取并验证的内容，才算真实可用知识。
5. 快速纠偏：小步变更、小步验证，允许快速修复，避免长时间冻结。

## 在本仓库的落地

- `AGENTS.md` 只做导航与硬约束，不堆砌细节。
- `docs/` 作为版本化知识库，记录架构、流程、计划与决策。
- `Cargo.toml` 统一 workspace 规则；crate 通过 `[lints] workspace = true` 继承。
- 质量门由 pre-commit + CI 双重执行，保持本地与远端一致。
